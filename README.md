# SpendSense

Personal finance API that categorises transactions using ML, tracks spending patterns, detects anomalies, and serves insights — all behind a clean pastel UI.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | Python 3.12, FastAPI, Pydantic, Uvicorn |
| Database | SQLite (dev) / PostgreSQL (prod), SQLAlchemy ORM |
| ML | scikit-learn (TF-IDF + Naive Bayes), NumPy |
| Auth | JWT (python-jose), bcrypt (passlib), OAuth2 password flow |
| Frontend | Vanilla HTML/CSS/JS, pastel pink theme |
| API Docs | Swagger UI (auto-generated by FastAPI) |

## Features

- **ML transaction categorisation** — TF-IDF vectoriser + Naive Bayes classifier trained on 190+ labelled examples. Falls back to regex keyword matching when confidence is low.
- **11 categories** — groceries, dining, transport, subscriptions, utilities, rent, shopping, health, entertainment, transfers, income
- **Manual override** — dropdown on every transaction to reassign category if ML gets it wrong
- **Monthly spending summary** — breakdown by category with percentages
- **Week-over-week trends** — spend totals with % change
- **Anomaly detection** — flags transactions > 2 standard deviations above category average
- **Dismiss anomalies** — clear false positives without deleting the transaction
- **Monthly history** — browse past months with full category breakdowns
- **JWT auth** — register, login, token-protected endpoints
- **CORS enabled** — ready for frontend integration or mobile apps

## Quick Start

```bash
# Clone
git clone https://github.com/jahnavibadam/personal-finance-api.git
cd personal-finance-api

# Install dependencies
pip install -r requirements.txt

# Seed demo data (creates demo user + 75 sample transactions)
python seed.py

# Run
uvicorn app.main:app --reload --port 8000
```

Open **http://localhost:8000** — redirects to login page.

## How Categorisation Works

1. **ML model** (primary) — TF-IDF extracts word features from description + merchant, Naive Bayes classifies into one of 11 categories. Trained on 190+ labelled examples covering UK spending patterns.

2. **Keyword fallback** — if ML confidence is below 40%, falls back to regex word-boundary matching against known brand names and terms.

3. **Manual override** — user can always reassign via the dropdown on the dashboard.

## How Anomaly Detection Works

For each category, the system computes the mean and standard deviation of all transaction amounts. Any transaction where `amount > mean + 2σ` gets flagged. Users can dismiss false positives.
